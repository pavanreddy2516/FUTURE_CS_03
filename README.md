# API Security Risk Analysis

## Overview

This repository contains my submission for **Task 3 – API Security Risk Analysis** as part of the **Future Interns Cyber Security Internship Program**.

The objective of this project was to perform a **read-only security assessment** of a public REST API using **Postman**. The analysis focused on understanding API behavior, inspecting HTTP requests and responses, reviewing response headers, identifying potential security risks, and documenting findings with appropriate recommendations.

The assessment was conducted on **JSONPlaceholder**, a publicly available REST API designed for learning and testing purposes. No exploitation, unauthorized access, or destructive testing was performed during this project.

---

# Objectives

1. Understand the architecture and functionality of REST APIs.
2. Analyze multiple API endpoints using Postman.
3. Inspect HTTP requests, responses, and response headers.
4. Evaluate authentication and authorization mechanisms.
5. Identify potential API security risks.
6. Assess the business impact of identified risks.
7. Recommend industry-standard security best practices.
8. Document the complete assessment in a professional security report.

# Methodology

1. Reviewed the API documentation.
2. Selected multiple public endpoints for assessment.
3. Sent HTTP GET requests using Postman.
4. Examined response data returned by the API.
5. Inspected HTTP response headers.
6. Checked for authentication and authorization mechanisms.
7. Identified potential security observations.
8. Classified risks based on severity.
9. Documented recommendations and mitigation strategies.

---

# Key Findings

1. Public endpoints were accessible without authentication.
2. API responses were returned in JSON format.
3. Standard HTTP response headers were properly configured.
4. No authentication mechanism was implemented because the API is intended for public testing.
5. No authorization checks were present for the public endpoints.
6. No exploitable vulnerabilities were identified within the approved assessment scope.
7. The API provides a suitable environment for learning REST API security concepts.
